제목 : 경북대 학식 도시락
주제 : 코로나 확산으로 인한 학식 도시락 서비스

주요 기능 
1. 모바일과 PC에 대응할 수 있는 반응형 웹서비스 제공
2. 저작권 문제가 없는 웹 디자인 활용
3. 웹브라우저 호환성 문제가 없는 코드 활용
4. MYSQL 활용하여 EC2 EB로 배포 가능하도록 구현

필요 기능
1. 도시락 메뉴 보여주기
: table 기반으로 하고, UI 디자인을 위해서 커스텀 하여 활용할 것.
2. 사용자 구매 기록
: 구매 기록이 당일 기준 2회 이상일 경우를 대비하여 1차적으로 표로 제공하고, 각각의 기록으로 들어가도록 제공
3. 구매 기록 QR 인증
: REST API를 활용하여 해당 기록의 해쉬 값을 비교하고 맞으면 제공
4. 로그인, 회원가입 제공
: SPRING 자체의 인증을 활용하여 제공, front는 아이디, 비밀번호를 SPRING에 fetch 형식으로 전달하여 토큰을 습득해 활용만 하는 것으로 통합하여 사용하는 방식으로 제공
5. 분리수거장소 지도 제공
: KAKAO MAP API key를 활용하여 제공
6. 경북대 생협 공지사항 제공
: 크롤링을 활용하여 DB에 미리 넣어두는 방식으로 제공. 12시간 or 24시간 마다 크롤링 고려
7. 자체 페이지 제공
: 이벤트 게시판, 문의 게시판 기능 제공
8. 마이페이지 제공
: 비밀번호 변경 및 본인 정보 제공
9. 관리자 페이지 제공
: 도시락 판매 통계, 이벤트 등록, 메뉴 남은 개수 관리 기능 제공
